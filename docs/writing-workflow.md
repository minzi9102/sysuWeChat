# 中大公众号新稿写作流程

本流程用于把活动原始材料与知识库中的写作经验组合成一篇新稿。知识库负责提供写法参考，不负责补充本次活动事实。

> **写作前事实边界（硬规则）**
>
> 参考库只提供结构、语气、句式、图文节奏和风险约束。本次稿件的人名、时间、地点、数字、项目、直接引语和结论，只能来自用户提供的活动材料。材料未说明的事实必须留空、标注待核或向用户确认，不得从参考文章迁移、拼接或推断。

## 一页流程

1. **输入活动材料**：收集通知、议程、采访记录、现场笔记、图片及已有审核口径，保留材料来源。
2. **判断任务类型**：明确稿件是活动报道、人物故事、成果发布、通知公告或其他类型，并提炼检索关键词。
3. **生成 context pack**：按任务类型和关键词检索参考文章、结构、风格、视觉模式与风险约束。
4. **建立事实清单**：只从活动材料整理人名、时间、地点、数字、项目、引语和结论；为缺失或冲突项标注待核。
5. **起草正文**：参考中大公众号的结构和语气组织活动事实，不照搬参考文章中的具体事实或事实性表达。
6. **风格改写**：调整标题、开头、段落节奏、过渡和结尾；删减空泛强调、套话及生硬的先否定后肯定句式。
7. **事实风险校验**：逐项回查事实清单；对“首次、首个、最大、最高、重磅”等强断言确认依据。
8. **输出发布包**：交付正文、备选标题、逐图图注和发布检查清单，并显式列出所有待核项。

## 使用 context pack

在仓库根目录运行：

```powershell
./scripts/make-writing-context.ps1 `
  -Root . `
  -ArticleTypes '活动报道' `
  -Keywords '活动主题,核心对象' `
  -Output 'writing_context/current.json'
```

各字段只用于写法选择：

| 字段 | 用途 |
| --- | --- |
| `reference_articles` | 选择相近稿件，观察整体结构和传播目标 |
| `structure_templates` | 选择标题、开头、过渡、正文和结尾骨架 |
| `reusable_styles` | 参考语气、修辞和表达方法，使用前结合本稿语境改写 |
| `visual_patterns` | 规划图片类型、顺序、叙事作用和图注节奏 |
| `risk_checklist` | 形成事实、引语、强断言和适用场景检查项 |

context pack 命中不足时，优先扩大关键词或调整任务类型；不要用不相干参考文章填补本次活动信息。

## 风格与发布检查

草稿完成后运行轻量风格检查：

```powershell
./scripts/lint-draft-style.ps1 -Path 'drafts/current.md'
```

lint 只提示需要复核的表达，不自动改写，也不会因命中提醒项而使命令失败。作者应结合上下文判断保留、改写或补充证据。

发布前确认：正文事实均可回链到活动材料；标题和图注没有新增事实；直接引语逐字核对；强断言有依据；人名、职务、时间、地点和数字一致；待核项已清零或明确标出。
